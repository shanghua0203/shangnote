# 安全

## Dirty Frag
**Dirty Frag：基於網路子系統的 Linux 核心權限提升漏洞**  
`dirtyfrag` 是一個利用 Linux 核心網路協議棧缺陷，達成非特權使用者對唯讀頁面快取 `(Page Cache)` 進行任意寫入的漏洞利用鏈。其效果類似於著名的 `Dirty Pipe (CVE-2022-0847)`，但觸發路徑位於網路層的 `sk_buff (Socket Buffer)` 碎片處理邏輯。

### 1. 核心
該 Exploit 通常結合了兩個主要漏洞以達成最大相容性：

- `CVE-2026-43284`：存在於核心處理網路分段 (Fragmentation) 時的邏輯錯誤。當核心在處理 `skb_shared_info` 結構中的 `frag_list` 時，未能正確校驗頁面引用計數或頁面屬性。

- `CVE-2026-43500`：涉及使用者命名空間 (User Namespace) 下的網路套接字權限繞過，允許無特權使用者觸發受損的內存映射邏輯。

###　2. 技術原理：從 skb 到 Page Cache核心概念在於 Zero-copy (零拷貝) 機制的濫用。Linux 為了優化效能，允許網路資料直接從頁面快取發送，而無需進行昂貴的 CPU 拷貝。  
1. **分頁映射 (Page Mapping)**：攻擊者首先打開一個唯讀文件（例如 /etc/passwd），該文件的內容會被加載到內核的 Page Cache 中。
2. **建構惡意 skb**：透過特定的網路系統調用，攻擊者在核心空間創建一個 `sk_buff` 結構，並利用漏洞將其碎片列表 (frag_list) 中的指標指向上述唯讀文件在 `Page Cache` 中的實體分頁。
3. **觸發寫入原語**：利用 `sendfile()` 或 `splice()` 系統調用，結合特定的核心路徑（如遺留的 ip_fragment 邏輯），強制核心將受控的資料流「注入」到這些本應是唯讀的頁面中。
4. **髒頁回寫 (Dirty Page Writeback)**：一旦 Page Cache 中的頁面被「弄髒」(Dirtying)，核心的寫回機制 (pdflush/kupdate) 最終會將這些惡意修改同步回磁碟上的實體文件。

### 3. 攻擊流
1. **環境準備**：進入一個新的 User Namespace 以獲得必要的網路配置權限（如修改路由或介面）。
2. **目標定位**：定位目標 SUID 程式或敏感配置文件（如 /etc/ld.so.preload）的磁碟偏移量。
3. **管道注入 (Pipe Injection)**：透過構造特殊的管道操作，將惡意 Payload（如提權 Shellcode）準備在記憶體中。
4. **觸發覆寫**：執行漏洞程式碼，利用核心網路組件將 Payload 覆蓋至唯讀頁面。
5. **提權**：執行已被篡改的 SUID 檔案，獲取 root 權限。

## 總結
`dirtyfrag` 的技術突破點在於它證明了網路分段邏輯與記憶體管理子系統之間的邊界檢查不足。與傳統的記憶體破壞漏洞（如堆疊溢位）不同，它屬於邏輯型漏洞，利用了核心對「分頁所有權」判斷的失效，使得攻擊者能穩定地控制系統中最核心的信任根源——文件系統。
